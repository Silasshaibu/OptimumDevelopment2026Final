# Homepage Recreation Brief (Premium Fintech Style)

## Project Goal
Recreate the current homepage with a cleaner, premium fintech look while preserving the same core business message: payment processing, merchant onboarding, product discovery, trust, and conversion.

## Primary Outcome
Build a high-conversion homepage that feels:
- Credible and enterprise-ready
- Modern and premium (not generic SaaS)
- Fast, clear, and mobile-first

## Target Audience
- Small and mid-sized business owners
- Retail, restaurant, and service operators evaluating payment/POS options
- Merchants comparing providers based on trust, cost savings, and device options

## Brand Positioning Direction
- Tone: confident, clear, practical, trustworthy
- Personality: premium fintech advisor (not sales-heavy, not flashy)
- Message pillar: reduce fees, modernize checkout, and simplify operations

## Information Architecture (Homepage Sections)
Keep this section order for continuity with current user expectations:
1. Utility promo banner (top strip carousel)
2. Main navigation + search + shop CTA
3. Hero carousel (core value propositions)
4. Micro-services slider (capabilities)
5. Top-selling products carousel
6. Handheld devices (featured + product grid)
7. Supplies (featured + product grid)
8. Social proof and ratings summary
9. Review submission module
10. Partner logo trust grid
11. Footer and legal/support links

## Visual Design Direction (Premium Fintech)
- Use a clean light base with deep navy/charcoal surfaces and controlled accent colors.
- Prioritize whitespace, stronger typographic hierarchy, and card depth.
- Keep imagery purposeful: devices, merchant environments, transaction moments.
- Replace loud color blocks with refined gradients, subtle glows, and soft shadows.

### Suggested Design Tokens
- Core background: #F5F8FC
- Surface: #FFFFFF
- Primary text: #0E1A2B
- Secondary text: #4E5D78
- Primary accent: #0A66FF
- Success accent: #17B26A
- Border: #D7E1EE
- Radius scale: 12px / 16px / 24px
- Shadow: soft layered shadows, low blur contrast

## Typography
- Headline style: strong geometric sans (600-700 weight)
- Body style: neutral sans (400-500 weight)
- Target reading feel: concise and executive, not verbose
- Ensure clear contrast for all text, especially over media backgrounds

## UX and Interaction Expectations
- Carousels should feel smooth and deliberate (no abrupt movement).
- CTA hierarchy must be consistent:
  - Primary CTA: solid fill
  - Secondary CTA: outline or subtle tinted surface
- Keep hover/focus states visible and premium (not overly animated).
- Maintain mobile parity for key actions (Apply, Learn More, Shop, Review).

## Section-by-Section Creative Guidance

### 1) Top Promo Banner
- Keep rotating promotional messages.
- Make the strip visually cleaner with tighter spacing and better contrast.
- Ensure clear click targets and obvious direction controls.

### 2) Navigation
- Retain current structure (Processing Solutions, Digital Services, Financing, etc.).
- Improve perceived quality with cleaner spacing, refined icon sizes, and stronger sticky behavior.
- Keep search instantly accessible on desktop and mobile.

### 3) Hero Carousel
- Keep 4-5 message pillars:
  - Payment Processing
  - Merchant Application
  - Business Financing
  - Digital Services
  - Brand/Company value slide
- Each hero slide should have one clear headline, one supporting sentence, and 1-2 CTAs.
- Use premium fintech visuals: device mockups, transaction UI overlays, subtle data motifs.

### 4) Micro-services
- Present as capabilities rather than generic promo tiles.
- Improve consistency across tile typography and image treatment.
- Add subtle icon overlays or metadata labels for a fintech-system feel.

### 5) Product Sections (Top Selling, Handheld, Supplies)
- Keep existing category structure and carousel/grid behavior.
- Standardize product card anatomy:
  - Product name
  - One-line value summary
  - Action (Learn More or View Product)
- Improve spacing and edge-to-edge rhythm so cards feel premium.

### 6) Reviews and Reputation
- Preserve aggregated rating and platform trust indicators.
- Highlight average rating and review count as a trust signal near CTAs.
- Keep “Write us a review” flow but style modal with premium fintech UI polish.

### 7) Partner Logos
- Keep logo wall but normalize visual weight and spacing.
- Use grayscale-to-color hover treatment only if subtle.
- Ensure logos read as validation, not decoration.

## Conversion Strategy
Primary conversion events:
- Merchant application starts
- Product detail visits
- Shop visits (supplies)
- Contact intent

Secondary conversion events:
- Review submissions
- Promo clicks
- Search usage

## Performance and Accessibility Requirements
- Maintain strong Lighthouse performance targets.
- Use responsive images and avoid oversized hero assets.
- Ensure keyboard access for sliders, modals, and menus.
- Meet WCAG contrast standards for all text/controls.
- Keep ARIA labels and semantic structure for all interactive components.

## Technical Constraints and Continuity
- Preserve the current PHP partial-driven assembly and route behavior.
- Maintain section-level modularity for easy CMS/database content updates.
- Keep existing core interactions unless redesign explicitly improves usability.

## Success Criteria
- Homepage feels visibly more premium and fintech-focused at first glance.
- Users can still navigate familiar categories and conversion paths quickly.
- Mobile experience is as complete and polished as desktop.
- Trust signals (reviews, partners, financing, product credibility) are stronger and easier to scan.

## Deliverables
1. High-fidelity homepage design (desktop + mobile)
2. Updated design tokens and component states
3. Interaction spec for carousels, CTAs, and review modal
4. Frontend implementation plan mapped to existing partial structure
